Overview
--------

Documentation about GCOOS-specific datasets at IMARS.  The aims of this
document are to provide:

* a xref:summary[summary] GCOOS-related files available on the IMARS FTP and THREDDS
servers;
* an xref:dataset_details[overview] of what is contained within each type of dataset file;
* xref:products[pointers] to where information about the products/variables within the
files can be found.

The majority of files are available via the FTP site:

ftp://imars.marine.usf.edu/[+ftp://imars.marine.usf.edu/+]

with a limited subset also available via a THREDDS server:

http://imars.marine.usf.edu:8080/thredds/catalog.html[+http://imars.marine.usf.edu:8080/thredds/catalog.html+]

The following xref:summary[summary] section lists the GCOOS-related subdirectories on the
FTP site, and adds a link to the THREDDS server when available.
Details about the datasets are available in the xref:dataset_details[dataset details] section.
Details about the provenance of the variables contained within the dataset
files can be found in the xref:products[products] section.


The total size of all daily, weekly, monthly and yearly final OC,
SST, SST4 and NPP files is under 0.5 Tb.  The intermediate files are
not included in this estimate.

-----
netcdf AQUA.20030101.0000.gcoos.oc {

	fakeDim0 = 1430 ;
	fakeDim1 = 2090 ;
	fakeDim2 = 1430 ;
	fakeDim3 = 2090 ;
	fakeDim4 = 1430 ;
	fakeDim5 = 2090 ;
	fakeDim6 = 1430 ;
	fakeDim7 = 2090 ;

variables:
	float chlor_a(fakeDim0, fakeDim1) ;
		chlor_a:Projection Category = "IDL" ;
		chlor_a:Projection Name = "Cylindrical" ;
		chlor_a:Source = "A2003001175000_1.L2b" ;
		chlor_a:units = "mg m^-3" ;
		chlor_a:Slope = 1.f ;
		chlor_a:Intercept = 0.f ;
		chlor_a:Scaling = "linear" ;
		chlor_a:Limit = 18., -98., 31., -79. ;
		chlor_a:Projection ID = 8 ;
		chlor_a:Latitude Center = 0. ;
		chlor_a:Longitude Center = -88.5 ;
		chlor_a:Rotation = 0. ;
		chlor_a:Position = 0., 0., 1., 1. ;
		chlor_a:Isotropic = 0 ;
		chlor_a:Scale = 0. ;
		chlor_a:Central_Azimuth = 0. ;

MODIS SST Fields
~~~~~~~~~~~~~~~~

The variable +SST+ is used for the SST derived from long-wave MODIS data.  The
details are given at:

http://oceancolor.gsfc.nasa.gov/DOCS/modis_sst/sst_transition.html[+http://oceancolor.gsfc.nasa.gov/DOCS/modis_sst/sst_transition.html+]

A brief synopsis:  "The long-wave SST algorithm makes use of MODIS bands 31
and 32 at 11 amd 12 um. The brightness temperatures are derived from the
observed radiances by inversion (in linear space) of the radiance versus
blackbody temperature relationship."

*Source Code*:
http://oceancolor.gsfc.nasa.gov/DOCS/OCSSW/sst_8c_source.html[+http://oceancolor.gsfc.nasa.gov/DOCS/OCSSW/sst_8c_source.html+]

MODIS SST4 Fields
~~~~~~~~~~~~~~~~

The variable +SST4+ is used for the SST derived from short-wave MODIS data.  The
details are given at:

http://oceancolor.gsfc.nasa.gov/DOCS/modis_sst/sst_transition.html[+http://oceancolor.gsfc.nasa.gov/DOCS/modis_sst/sst_transition.html+]

A brief synopsis: "The short-wave SST algorithm makes use of MODIS bands 22
and 23 at 3.959 amd 4.050 um. The brightness temperatures are derived from the
observed radiances by inversion (in log space) of a the radiance versus
blackbody temperature relationship."

*Source Code*:
http://oceancolor.gsfc.nasa.gov/DOCS/OCSSW/sst_8c_source.html[+http://oceancolor.gsfc.nasa.gov/DOCS/OCSSW/sst_8c_source.html+]
